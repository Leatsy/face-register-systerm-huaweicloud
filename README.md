前端页面设计
有以下几个界面：
1. 用户从手机端首先进入之后看到的界面至少有以下内容：处于主要位置的“签到”按钮（用于跳转到签到选择界面，跳转无需处于登录状态）和“签到发布”按钮（跳转到签到发布界面；签到发布需要处于登录状态），和右上角的登录按钮（用于跳转到登录界面；如果用户已经登录，就变成用户的头像，头像图片使用用户注册时上传的图片）

2. 签到选择界面，按时间顺序列出目前尚未过期的签到事件（每个事件用简约的卡片展示；过期的不展示，未过期的但未开始的也不展示），事件需要包含事件名称、事件开始时间、事件结束时间、事件描述等信息。通过点击事件可以跳转到签到界面。
3. 签到发布界面，用户可以在这个界面发布新的签到事件，包括事件名称、事件开始时间、事件结束时间、事件描述等信息。开始时间默认使用当前时间，且不可早于当前时间；结束时间默认使用当前时间加1小时。
4. 签到界面，用户可以在这个界面进行签到操作，包括上传照片（可以使用相机拍照，也可以从相册中选取图片）、确认签到等。确认签到后，系统会根据上传的照片检索到对应的用户，标记其签到成功，并且前端出现签到成功的提示信息并自动跳转到第一个界面；如果检索到的用户不存在，前端出现签到失败的提示信息。

5. 登录界面，用户可以在这个界面输入用户名和密码进行登录。登录成功后跳转第1条的界面（用户可以在右上角看到自己的头像，通过点击头像可以查看个人信息界面）。在这个登录界面也有注册按钮，用户可以点击跳转到注册界面。
6. 个人信息界面，用户可以在这个界面查看自己的基本信息，包括学号、姓名、手机号、照片等。用户可以在界面查看自己的签到记录。在这个页面的最下方有注销功能，用户可以点击注销登录。
7. 注册界面，用户可以在这个界面输入用户名、密码、确认密码、姓名、手机号、上传标准人脸照片等信息进行注册。注册成功后自动登录并跳转到第1条的界面。
